# MolForge — Complete Setup Guide
# ================================
# Read this FULLY before starting.
# Every step is explained for beginners.

===========================================================
  WHAT THIS PROJECT IS
===========================================================

You built an AI system that:
  1. Takes a protein pocket (.pdb file) as input
  2. Runs the PMDM dual diffusion model to generate drug molecules
  3. Shows the molecules in 3D with drug-likeness scores
  4. Lets you download the generated .sdf files

The "frontend" is the website people see.
The "backend" is the Python server that does the AI work.


===========================================================
  YOUR FOLDER STRUCTURE
===========================================================

  MolForge/
  ├── app.py                  ← Backend server (the brain)
  ├── requirements.txt        ← Python packages needed
  ├── web/
  │   └── index.html          ← Frontend website (what users see)
  ├── generated/
  │   └── source/             ← Put your .sdf files here for demo
  ├── uploads/                ← Where uploaded .pdb files go (auto)
  └── jobs/                   ← Where results are stored (auto)


===========================================================
  STEP 1: INSTALL PYTHON (if you don't have it)
===========================================================

1. Go to https://www.python.org/downloads/
2. Download Python 3.11 (click the big yellow button)
3. Run the installer
   !! IMPORTANT: Check "Add Python to PATH" before clicking Install !!
4. Open Command Prompt (Windows) or Terminal (Mac/Linux)
5. Type: python --version
   You should see: Python 3.11.x
   If yes, Python is installed correctly.


===========================================================
  STEP 2: SET UP THE PROJECT FOLDER
===========================================================

1. Create a folder on your Desktop called "MolForge"
2. Copy these files into it:
     app.py
     requirements.txt
     web/ (the whole folder with index.html inside)
     generated/source/ (create this folder)

3. Open Command Prompt or Terminal
4. Navigate to your folder:
   Type: cd Desktop\MolForge      (Windows)
   Type: cd ~/Desktop/MolForge    (Mac/Linux)

5. You should now be inside the MolForge folder.
   Check by typing: dir (Windows) or ls (Mac/Linux)
   You should see app.py listed.


===========================================================
  STEP 3: INSTALL REQUIRED PACKAGES
===========================================================

In your Command Prompt / Terminal, type exactly:

  pip install fastapi uvicorn python-multipart

Then for RDKit (needed to compute molecule metrics):

  pip install rdkit

  If rdkit fails, try:
  pip install rdkit-pypi

Wait for it to finish (may take 2-3 minutes).
You should see "Successfully installed" at the end.


===========================================================
  STEP 4: GET YOUR SDF FILES (DEMO DATA)
===========================================================

Your .sdf files are in Google Drive from when you ran Notebook 3.

Path on Drive: PMDM/dataset/sdf_files/

How to download them:
1. Go to Google Drive in your browser
2. Navigate to: MyDrive → PMDM → dataset → sdf_files
3. You'll see folders like "2VUK_CDK2", "7L11_Mpro", etc.
4. Open any folder and download 3-6 .sdf files
5. Put them inside: MolForge/generated/source/

Example: Copy "2VUK_0.sdf", "2VUK_1.sdf", "2VUK_2.sdf"
         into MolForge/generated/source/

!! You need at least 1 .sdf file in generated/source/ for demo mode !!


===========================================================
  STEP 5: START THE BACKEND SERVER
===========================================================

In Command Prompt / Terminal (inside MolForge folder):

  python app.py

You should see:
  ================================================
    PMDM Backend starting...
    Open http://localhost:8000 in your browser
  ================================================

The server is now running.
DO NOT close this window — it needs to stay open.

If you see an error about port 8000 being in use:
  python app.py --port 8001
  Then open http://localhost:8001 instead


===========================================================
  STEP 6: OPEN THE FRONTEND IN YOUR BROWSER
===========================================================

1. Open Google Chrome (or any browser)
2. Type in the address bar: http://localhost:8000
3. Press Enter
4. You should see the MolForge website!

If it says "404" or blank:
  - Make sure the server is still running (check your terminal)
  - Make sure web/index.html exists in your MolForge folder
  - Try: http://localhost:8000/ui/


===========================================================
  STEP 7: TEST THE DEMO
===========================================================

1. You need a .pdb file to upload. Get one here:
   https://files.rcsb.org/download/2VUK.pdb
   (just paste that link in your browser — it downloads automatically)

2. In the MolForge website:
   a. Click "Drop .pdb file here" and select 2VUK.pdb
   b. Set "Atom budget" to 20
   c. Set "Molecules to generate" to 3
   d. Click "Generate Molecules"

3. Wait 5-10 seconds
4. You should see 3D molecules appear with their QED scores!

!! If nothing appears, check the next section !!


===========================================================
  TROUBLESHOOTING
===========================================================

Problem: "No molecules generated"
Fix: Make sure you have .sdf files in MolForge/generated/source/
     Download them from Drive as explained in Step 4.

Problem: "Connection refused" when opening http://localhost:8000
Fix: The server isn't running. Go to your terminal and run: python app.py

Problem: "pip is not recognized"
Fix: Python isn't installed or not in PATH. Reinstall Python from
     python.org and make sure to check "Add to PATH" during install.

Problem: RDKit install fails
Fix: Try: pip install rdkit-pypi
     Or: conda install rdkit (if you have Anaconda)

Problem: Port 8000 already in use
Fix: python app.py --port 8001
     Then open http://localhost:8001

Problem: "Module not found" error
Fix: Run: pip install fastapi uvicorn python-multipart rdkit

===========================================================
  ACCURACY — IS IT OK?
===========================================================

Typical results you might see:

  Experiment 1 (Random Forest on CrossDocked):  R² = 0.3–0.6
  Experiment 2 (Your GNN on CrossDocked):       R² = 0.4–0.7
  Experiment 3 (Train yours, test CrossDocked): R² = 0.2–0.5
  Experiment 4 (Train CrossDocked, test yours): R² = 0.4–0.6

Any R² > 0.4 is acceptable for this task (QED prediction from molecular
structure is genuinely hard). The key story is:
  - Your GNN does better than Random Forest (Exp 2 > Exp 1)
  - Training on PMDM-generated data still generalizes (Exp 3)
  - This validates your entire pipeline


===========================================================
  PROJECT ARCHITECTURE (what to explain)
===========================================================

Notebook 1: Set up environment, downloaded pretrained PMDM checkpoint
Notebook 2: Ran PMDM on 2VUK pocket → generated 3 molecules (proof of concept)
Notebook 3: Generated 300+ molecules across 30 protein targets → dataset
            Trained our own GNN on that dataset → best_model.pt
Notebook 4: 4 experiments comparing models and datasets
Frontend  : MolForge UI — upload PDB → generate → view in 3D → download

The full pipeline: protein .pdb → PMDM dual diffusion → 3D molecules → 
                   GNN ranking by drug-likeness → frontend display

2. cd Desktop\MolForge   (or wherever you put the folder)
3. python app.py
4. Open http://localhost:8000 in your browser
